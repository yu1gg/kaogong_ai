# 使用与维护与测试说明

这份文档面向项目的开发者与维护者，重点回答 3 件事：

1. 这个项目在本地应该怎么启动和验证。
2. 改动不同模块后，应该重跑哪些维护动作与回归流程。
3. 当前测试体系覆盖了什么、默认怎么跑、哪些环境问题需要单独注意。

如果你想先看项目全貌，请先读 [README.md](../README.md)；如果你要深入看回归与标定参数，请继续读 [回归与标定说明.md](回归与标定说明.md)。

## 1. 适用读者与文档边界

这份文档默认你正在维护仓库根目录 `kaogong_ai/` 下的代码与数据，适合以下场景：

1. 本地启动 FastAPI 服务并做日常调试。
2. 调整题库、评分规则、Prompt、回归脚本或导入脚本。
3. 判断某次改动后应该跑哪些测试、哪些回归、哪些人工检查。

这份文档不负责：

1. 详细解释产品背景或业务目标。
2. 完整展开每个回归脚本的所有参数细节。
3. 生产部署、上线运维或 CI/CD 方案设计。

## 2. 项目结构与目录职责

下面这些目录最值得先建立清晰边界：

| 路径 | 角色 | 建议维护方式 | 备注 |
| --- | --- | --- | --- |
| `ai_gongwu_backend/app/` | 后端运行代码 | 人工维护 | FastAPI、依赖注入、评分链路、媒体处理、LLM 调用都在这里 |
| `ai_gongwu_backend/assets/questions/` | 题库入口目录 | 人工 + 生成混合 | 根目录手工题目可人工维护，`generated_hunan/` 是脚本产物 |
| `ai_gongwu_backend/assets/questions/generated_hunan/` | 湖南自动生成题库 | 脚本生成 | 不建议逐个手工修改，优先改导入脚本后重生成 |
| `ai_gongwu_backend/assets/regression_samples/generated_hunan/` | 自动生成回归样本 | 脚本生成 | 与题目 JSON 中的 `regressionCases[*].sample_path` 对应 |
| `reports/regression/` | 回归与标定报表输出目录 | 脚本生成 | `run_regression.py` 与 `run_llm_regression.py` 写入此处 |
| `sample_sets/` | 早期人工样本分类参考 | 人工维护 | 主要用于理解样本层次，不是当前批量回归的直接输入 |
| 根目录 `*.docx` / `*.extracted.txt` | 湖南题库源文档与提取文本 | 人工维护 | `import_hunan_question_bank.py` 会读取这些源文件 |
| 根目录历史样本文本 | 历史评分样本 | 人工维护 | 如 `26分.txt`、`标准1.txt` 等，当前仍被部分测试引用 |
| `docs/` | 项目文档 | 人工维护 | 本文档、变更记录、回归说明都在这里 |

当前项目里最容易混淆的几件事：

1. `sample_sets/` 不是当前批量回归的样本源。
2. `generated_hunan/` 下的题目和样本是生成产物，不是长期手工编辑入口。
3. 根目录历史样本文本不是废弃文件，`test_scoring_calculator.py` 仍会直接读取它们。

## 3. 环境准备与配置

### 3.1 Python 与依赖

仓库根目录的 `.python-version` 当前固定为 `3.10`，建议用 Python 3.10 运行。

从仓库根目录初始化环境的推荐方式：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r ai_gongwu_backend/requirements.txt
```

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r ai_gongwu_backend/requirements.txt
```

如果你已经使用 conda 或其他解释器环境，只要确保安装的是 `ai_gongwu_backend/requirements.txt` 中的依赖即可。

### 3.2 系统依赖

当前媒体链路依赖以下系统或大型组件：

1. `ffmpeg`
2. `torch`
3. `openai-whisper`
4. `opencv-python-headless`

几个关键差异要记住：

1. 缺少 `ffmpeg` 时，视频上传会直接失败。
2. `openai-whisper` 首次加载本地模型时，可能会自动下载权重。
3. 缺少 `opencv-python-headless` 时，视觉分析会被跳过，但文本 / 音频主链路不一定因此中断。

### 3.3 `.env` 位置与关键配置

项目会自动读取 `ai_gongwu_backend/.env`。仓库里当前没有 `.env.example`，需要手工创建。

一个最小可用示例：

```env
LLM_PROVIDER=QWEN
LLM_API_KEY=
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen3-coder-plus

QUESTION_DB_PATH=assets/questions
DATABASE_URL=sqlite:///storage/ai_gongwu.db

WHISPER_MODEL_SIZE=base
WHISPER_CPU_THREADS=4
WHISPER_LANGUAGE=zh
ENABLE_VISUAL_ANALYSIS=true
```

维护时最需要注意这几个配置项：

1. `LLM_API_KEY`
   为空时，系统不会初始化真实 LLM 客户端，评分链路会回退到确定性评分。
2. `QUESTION_DB_PATH`
   默认是 `assets/questions`，既支持指向单个 JSON，也支持指向题库目录。
3. `DATABASE_URL`
   默认落在 `ai_gongwu_backend/storage/ai_gongwu.db`。
4. `ENABLE_VISUAL_ANALYSIS`
   关闭后，视频链路不会做轻量视觉分析。

补充说明：

1. `QUESTION_DB_PATH` 和 `DATABASE_URL` 的相对路径会优先按 `ai_gongwu_backend/` 根目录解析。
2. 如果你换了启动目录，但仍使用相对路径，优先以 `app/core/config.py` 里的 `Settings.resolve_path()` 规则为准。

## 4. 启动与日常使用

### 4.1 推荐启动方式

方式一：在 `ai_gongwu_backend/` 内直接用 Uvicorn 启动。

```bash
cd ai_gongwu_backend
uvicorn app.main:app --reload --port 9000
```

如果当前环境里 `uvicorn` 不在 PATH，可改用：

```bash
cd ai_gongwu_backend
python -m uvicorn app.main:app --reload --port 9000
```

方式二：在 `ai_gongwu_backend/` 内用模块方式启动。

```bash
cd ai_gongwu_backend
python -m app.main
```

方式三：从仓库根目录直接运行入口文件。

```bash
python ai_gongwu_backend/app/main.py
```

补充说明：

1. `ai_gongwu_backend/run.sh` 只是一个 Unix 风格包装脚本，本质仍是 `uvicorn app.main:app --reload`。
2. 当前 `app/main.py` 已兼容直接运行脚本的场景。
3. 三种方式都应该最终提供同一套接口。

### 4.2 启动后最小验证顺序

建议每次本地改动后，至少按下面顺序做一次烟雾验证：

1. 打开 `http://127.0.0.1:9000/docs`
   目的：确认 FastAPI 服务已启动，Swagger 可访问。
2. 访问 `http://127.0.0.1:9000/health`
   目的：确认服务、题库加载、数据库初始化都没明显报错。
3. 访问 `GET /api/v1/interview/questions`
   目的：确认题库目录可读、题目 JSON 可正常加载。
4. 在 Swagger 中测试 `POST /api/v1/interview/evaluate/text`
   目的：确认文本旁路评分链路可用。
5. 如改动了媒体或转写链路，再测试 `POST /api/v1/interview/evaluate`
   目的：确认视频 / 音频路径、`ffmpeg` 与 Whisper 环境可用。

### 4.3 主要接口与日常用途

| 接口 | 主要用途 | 日常场景 |
| --- | --- | --- |
| `GET /health` | 服务基础健康检查 | 启动后第一时间确认服务是否活着 |
| `GET /api/v1/interview/questions` | 查看题目摘要列表 | 确认题库加载与题目数量是否正常 |
| `GET /api/v1/interview/questions/{question_id}` | 查看单题完整配置 | 核对题目 JSON、分档、样本配置 |
| `POST /api/v1/interview/evaluate/text` | 文本旁路评分 | 调 Prompt、调规则、快速回归 |
| `POST /api/v1/interview/evaluate` | 音频 / 视频评分 | 验证媒体解析、ASR、视觉分析 |
| `GET /api/v1/interview/records` | 查看最近测评记录 | 检查持久化是否正常 |
| `GET /api/v1/interview/records/{record_id}` | 查看单条记录详情 | 复盘 Prompt、原始输出、最终结果 |

## 5. 日常维护工作流

建议把“改了什么，就重跑什么”固定成下面这张表：

| 你改了什么 | 必做动作 | 建议补充验证 |
| --- | --- | --- |
| `app/services/scoring/prompts.py`、`app/services/scoring/calculator.py`、评分规则 | 先跑单元测试，再跑 `python scripts/run_regression.py` | 如果真实 LLM 行为也会受影响，再跑 `python scripts/run_llm_regression.py --repeat 3` |
| `scripts/import_hunan_question_bank.py`、样本生成逻辑、湖南源文档 | 先重新导入，再跑确定性回归，再跑 LLM 回归 | 子集稳定后才考虑 `--writeback` |
| 手工题目 JSON | 重新检查题目列表、题目详情与相关回归样本 | 如修改了 `regressionCases`，同步跑对应题目的回归 |
| 数据库配置、持久化逻辑、`evaluation_store.py` | 启动服务并测试 `/evaluate/text` 与 `/records` | 必要时备份 `ai_gongwu_backend/storage/ai_gongwu.db` |
| 媒体解析、Whisper、视频分析 | 用真实音频 / 视频做冒烟测试 | 确认 `ffmpeg`、Whisper 权重、OpenCV 都可用 |
| 启动入口、依赖注入、`config.py` | 重新验证三种启动方式与 `/health` | 额外检查题库路径和数据库路径是否仍可解析 |

### 5.1 题库与样本的正确维护方式

优先人工维护的地方：

1. `ai_gongwu_backend/assets/questions/` 根目录下的手工题目。
2. 根目录题库源文档与 `.extracted.txt`。
3. `docs/` 与 `sample_sets/`。

不建议逐个手工改的地方：

1. `ai_gongwu_backend/assets/questions/generated_hunan/`
2. `ai_gongwu_backend/assets/regression_samples/generated_hunan/`

正确姿势是：

1. 先改生成逻辑或源数据。
2. 再运行 `python scripts/import_hunan_question_bank.py`。
3. 然后跑回归，确认生成结果合理。

### 5.2 导入、回归、标定的推荐顺序

如果你的改动涉及题库、样本、评分规则或 LLM 标定，推荐固定顺序如下：

```bash
cd ai_gongwu_backend
python scripts/import_hunan_question_bank.py
python scripts/run_regression.py
python scripts/run_llm_regression.py --repeat 3
```

只有当子集结果稳定时，才建议进一步使用：

```bash
cd ai_gongwu_backend
python scripts/run_llm_regression.py --question-id <question_id> --repeat 3 --writeback
```

更完整的标定策略和参数说明，请看 [回归与标定说明.md](回归与标定说明.md)。

## 6. 测试与验证矩阵

### 6.1 默认单元测试入口

当前仓库的测试默认是 `unittest` 风格，不是 `pytest` 风格。

推荐命令：

```bash
cd ai_gongwu_backend
python -m unittest discover -s tests -p "test_*.py"
```

当前 `ai_gongwu_backend/requirements.txt` 没有包含 `pytest`，所以不要把 `pytest` 当成默认必备前提。

### 6.2 三组现有测试各覆盖什么

| 测试文件 | 主要覆盖内容 | 维护价值 |
| --- | --- | --- |
| `tests/test_question_bank.py` | 题库目录加载、题目查询、自动导入题目的元数据暴露 | 判断题目 JSON 与题库加载器是否稳定 |
| `tests/test_scoring_calculator.py` | 证据抽取校验、后处理修正、规则型评分、分档分离能力 | 判断评分链路核心逻辑是否被改坏 |
| `tests/test_evaluation_store.py` | 测评结果写库、列表查询、详情回读 | 判断数据库持久化与记录结构是否正常 |

当前测试还没有系统覆盖：

1. FastAPI 路由级接口测试。
2. `scripts/` 下导入 / 回归脚本的自动化测试。
3. 真实 LLM 的可重复 stub 或端到端 HTTP 测试。

### 6.3 冒烟验证与回归验证怎么分工

两类验证不要混淆：

1. 接口烟雾验证
   重点是服务能不能起来、健康检查能不能过、题库能不能读、文本测评能不能走通。
2. 回归脚本验证
   重点是分档排序、回归样本区间、LLM 波动与标定结果是否稳定。

推荐最小组合：

1. 每次改服务入口、配置、依赖注入后，先做接口烟雾验证。
2. 每次改评分逻辑、题库、样本生成逻辑后，再做回归验证。

### 6.4 回归脚本的日常用法

确定性 / 链路回归：

```bash
cd ai_gongwu_backend
python scripts/run_regression.py
```

真实 LLM 回归：

```bash
cd ai_gongwu_backend
python scripts/run_llm_regression.py --repeat 3
```

几个默认建议：

1. `run_regression.py` 适合先体检基础排序，不需要真实 LLM 密钥。
2. `run_llm_regression.py` 需要真实 `LLM_API_KEY`，且更适合在改 Prompt、换模型、调标定区间后执行。
3. 正式看 LLM 稳定性时，优先固定 `--repeat 3`。
4. 批量调整前，先用 `--question-id` 跑子集。

所有报表默认写入 `reports/regression/`。

### 6.5 当前观察到的环境限制

下面这些不是当前业务逻辑的确定性 bug，而是这次在受限 Windows 环境里观察到的运行限制：

1. `test_question_bank.py` 使用 `TemporaryDirectory()` 时，系统临时目录可能因权限限制导致创建 / 清理失败。
2. `test_evaluation_store.py` 创建临时 SQLite 文件时，可能因为临时目录权限或文件句柄限制报错。

如果你在类似环境里遇到这类问题，优先检查：

1. 当前解释器是否有可写的临时目录。
2. SQLite 测试文件是否能正常创建。
3. 是否需要在更正常的本地开发环境或可写工作目录下重跑测试。

## 7. 常见问题与排障

### 7.1 没有配置 `LLM_API_KEY`

现象：

1. 日志里提示 LLM 客户端未初始化。
2. 评分流程回退到确定性评分。
3. `run_llm_regression.py` 会直接无法按“真实 LLM 回归”预期工作。

处理方式：

1. 检查 `ai_gongwu_backend/.env` 是否存在。
2. 检查 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL_NAME` 是否正确。

### 7.2 没装 `ffmpeg`

现象：

1. 视频上传时直接报错。
2. `process_video()` 无法提取音频。

处理方式：

1. 安装系统级 `ffmpeg`。
2. 重新打开终端后再启动服务。

### 7.3 媒体后缀不受支持

现象：

1. `POST /api/v1/interview/evaluate` 返回 `400`。
2. 错误信息包含“不支持的媒体格式”。

当前支持的默认后缀：

1. 视频：`.mp4`、`.avi`、`.mov`、`.mkv`
2. 音频：`.mp3`、`.wav`、`.m4a`

如果你要新增格式，需要同时检查：

1. `app/core/config.py`
2. `InterviewFlowService.validate_media_suffix()`
3. 实际依赖链是否真的能处理该格式

### 7.4 题库路径或数据库路径找不到

现象：

1. 启动时报题库为空、题库路径不存在或数据库打不开。
2. 换了启动目录后，相对路径行为与你预期不一致。

处理方式：

1. 优先按 `ai_gongwu_backend/` 为基准理解相对路径。
2. 检查 `QUESTION_DB_PATH` 与 `DATABASE_URL` 是否写对。
3. 必要时改成绝对路径，或回到 `ai_gongwu_backend/` 再启动。

### 7.5 Whisper 或 OpenCV 相关依赖问题

现象：

1. `openai-whisper` 缺失时，音频转写无法进行。
2. Whisper 首次运行可能较慢，因为会初始化模型并可能下载权重。
3. `opencv-python-headless` 缺失时，视觉分析会被跳过。

处理方式：

1. 确认虚拟环境已正确安装 `requirements.txt`。
2. 对首次下载 Whisper 权重的场景预留更多时间。
3. 如果只调文本链路，可先用 `/evaluate/text` 做验证。

### 7.6 当前 README 中仍有历史绝对路径示例

现象：

1. `README.md` 和部分旧文档仍保留一些 `/home/quyu/...` 风格示例。

处理方式：

1. 日常开发、维护、测试时，以本手册中的相对路径命令为准。
2. 遇到专项流程，再回到相应专项文档查看细节说明。
