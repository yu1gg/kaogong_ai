## 第 1 轮：后端结构梳理与幻觉抑制增强

### 修改时间

2026-04-04

### 这一轮改动的目标

1. 修复后端中影响稳定性和可扩展性的核心问题。
2. 提升大模型评分结果的结构稳定性。
3. 降低“模型编造原话”“总分不一致”“维度乱写”等问题。
4. 把代码补成更适合弱基础学习者阅读的版本。

### 发现的主要问题

#### 1. 题目 ID 没有真正参与评分

虽然接口里传了 `question_id`，但原有流程实际上没有基于这个 ID 选择题目，导致接口参数与内部逻辑脱节。

#### 2. 视觉描述和考生原文被混在一起

原先视频处理后的文本把“视觉分析结论”和“语音转录内容”拼到一个字符串中直接送给大模型，容易导致模型把视觉观察误认为考生真的说过的话。

#### 3. 大模型输出缺少强校验

主要风险包括：

1. 返回未定义维度。
2. 分数超出满分。
3. 总分和分项得分不一致。
4. 引用了 transcript 中不存在的原话。

#### 4. 配置与路径解析不够稳定

项目在不同启动目录下运行时，题库路径可能出问题。

#### 5. 工程可维护性不够

1. 缺少完整的数据模型。
2. 依赖注入不集中。
3. 阅读门槛较高。

## 第 3 轮：河南省直定制评分校准

### 修改时间

2026-04-07

### 修改目标

1. 修复 `26分.txt` 这类“通用高分样本”被河南省直定制规则直接打入低分区的问题。
2. 保留“河南省情 + 省直岗位化”扣分能力，但避免重复放大。
3. 让接口层与真实评分链路回归验证通过。

### 修改内容

1. 调整第二阶段评分 Prompt，明确“先评通用内容质量，再评河南省直适配度”。
2. 在 Prompt 中加入分档锚点：
   - `24-30` 为河南省直高分
   - `20-23` 为通用高分但本土化不足
   - `16-19` 为基本合格
   - `0-15` 为明显偏弱
3. 收紧自动缺失证据规则：
   - 省情缺失和岗位视角缺失合并为一类缺失证据
   - 措施层次不足只在措施证据本身也不足时才补充
   - 口语化自动证据数量增加上限，避免重复刷负面证据
4. 在后处理增加“通用高分保底校准”：
   - 当答案已覆盖现象、危害、措施三大核心内容
   - 但仅缺河南省情/省直岗位化表达时
   - 若模型把它直接打入低分区，则回调到中上档
5. 新增针对该逻辑的单元测试。
