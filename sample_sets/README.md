# 样本分类说明

## 使用前先看

这个目录是项目早期整理的“河南样本人工分类”。

它和当前真实回归集不是一回事：

1. 这里主要用于人工理解“通用高分 / 河南省直高分 / 非高分参考”的区别。
2. 当前脚本真正批量回归使用的是 `ai_gongwu_backend/assets/regression_samples/generated_hunan/`。
3. `sample_sets/` 不会被 `run_regression.py` 或 `run_llm_regression.py` 直接自动读取。

所以：

1. 如果你要做规则理解、Prompt 对照，可参考这里。
2. 如果你要做批量回归、LLM 标定、区间回写，请以 `generated_hunan` 为准。

## 分类标准

### 通用高分

满足以下特征中的大部分：

1. 能完整覆盖积极意义、问题分析、改进措施。
2. 结构清楚，论证较完整。
3. 有一定治理思维和执行思维。
4. 但不一定明显体现“河南省情 + 省直机关岗位视角”。

### 河南省直高分

在“通用高分”基础上，进一步满足以下特征中的大部分：

1. 明确站在省直机关或省级统筹视角作答。
2. 明确结合河南省情、发展格局、省委精神或典型地方案例。
3. 用词明显更接近政策表达和机关表达。
4. 不只是说“因地制宜”，而是能具体落到省级分类指导、统筹协调、配套保障。

### 非高分参考

以下情况之一会归入这一组：

1. 虽然有一定分析，但本土化和岗位化明显不足。
2. 结构、措施、术语、表达水平均未达到高分稳定区间。
3. 更适合拿来做中低分对照，而不是高分标杆。

## 为什么这样分

### `26分.txt`

优点是通用综合分析能力较强，问题分析和改进措施完整，但缺少明显的河南省情、省直岗位职责、省委精神表达，所以更适合作为“通用高分”。

### `28分.txt`

整体结构、政策语言、治理逻辑都比较成熟，属于高质量通用答案。但它没有像省直标杆答案那样系统嵌入河南发展格局和省级统筹语境，所以仍放在“通用高分”。

### `标准1.txt`

这是明显的河南省直高分样本，省情、岗位视角、统筹协调、分类指导、案例、术语都比较完整。

### `中高分1.txt`

同样明显带有河南省直机关视角，虽然密度略低于 `标准1.txt`，但已经达到“河南省直高分样本”的标准。

### `中低分1.txt`

虽然提到了“河南”，也有一定本土化意识，但整体政策密度、岗位表达、统筹深度还不足以作为“河南省直高分”样本。

## 使用建议

1. 做通用评分校准时，优先用 `sample_sets/通用高分/`。
2. 做河南省直定制评分校准时，优先用 `sample_sets/河南省直高分/`。
3. 做分档能力验证时，再把 `sample_sets/非高分参考/` 一起加入。
